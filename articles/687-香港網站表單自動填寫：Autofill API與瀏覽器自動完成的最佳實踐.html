香港網站表單自動填寫的常見迷思

迷思一：自動填寫只需要設定name屬性

許多開發者認為只要給input元素設定了合適的name屬性，瀏覽器就能自動識別並填寫。實際上，現代瀏覽器主要依賴autocomplete屬性來判斷欄位用途。例如，autocomplete="street-address"告訴瀏覽器這是街道地址欄位，autocomplete="cc-number"表示信用卡號碼。不設定或錯誤設定autocomplete屬性會導致自動填寫功能完全失效。

迷思二：自動填寫會帶來安全風險

有些企業擔心啟用自動填寫會導致用戶敏感資訊被惡意網站竊取。實際上，瀏覽器的自動填寫功能有嚴格的安全機制——只在HTTPS加密頁面上生效、只在用戶明確選擇時才填入資訊、不會向第三方網站洩露用戶資料。不啟用自動填寫反而會讓用戶因為輸入繁瑣而使用弱密碼或在多個網站間重複使用相同密碼，這才是真正的安全風險。

迷思三：自訂輸入元件也能支援自動填寫

許多設計精美的網站使用自訂的div-based輸入元件替代原生input元素，期望通過JavaScript模擬輸入行為。這種做法會完全破壞瀏覽器的自動填寫功能——瀏覽器只能識別原生表單元素。2026年的最佳實踐是使用原生input元素並通過CSS進行樣式化，而非使用自訂元件替代。

迷思四：行動裝置不需要自動填寫

恰恰相反，行動裝置比桌面更需要自動填寫。在手機上輸入長地址、信用卡號等資訊極其痛苦，用戶放棄率更高。2026年的移動瀏覽器（如Chrome Mobile、Safari Mobile）都提供了強大的自動填寫功能，包括相機掃描信用卡、生物辨識確認填寫等。不配置自動填寫等於讓移動用戶承受最大的輸入摩擦。

香港網站表單自動填寫的7大最佳實踐

1. 正確設定autocomplete屬性

每個表單欄位都應設定正確的autocomplete屬性值。常用的值包括：name（全名）、given-name（名）、family-name（姓）、email（電郵）、tel（電話）、street-address（街道地址）、address-level1（地區）、postal-code（郵編）、country（國家）、cc-name（信用卡持卡人姓名）、cc-number（信用卡號）、cc-exp（到期日）、cc-csc（安全碼）。使用標準的autocomplete值是自動填寫功能的基礎。

2. 使用原生表單元素

始終使用原生input、select、textarea元素而非自訂div元件。瀏覽器的自動填寫功能只能識別原生表單元素。如果需要自訂樣式，使用CSS對原生元素進行樣式化，而非使用JavaScript模擬輸入行為。

3. 合理的表單結構與分組

使用fieldset和legend對相關欄位進行分組，幫助瀏覽器更好地理解表單結構。例如，將地址相關的欄位（街道、城市、郵編）放在同一個fieldset中，有助於瀏覽器將整個地址作為一組資訊填寫。

4. 避免JavaScript干擾自動填寫

某些JavaScript邏輯會干擾瀏覽器的自動填寫——例如即時格式化輸入值、動態切換input類型、在輸入時觸發重新渲染等。應確保JavaScript邏輯不會阻止或干擾瀏覽器填入值。對於需要格式化的欄位（如電話號碼），應在blur事件而非input事件時進行格式化。

5. 提供清晰的label關聯

每個input元素都應有關聯的label元素，通過for和id屬性建立關聯。這不僅是無障礙設計的要求，也幫助瀏覽器更好地理解欄位用途。某些瀏覽器會使用label文字作為自動填寫判斷的輔助資訊。

6. 支援Payment Request API

對於結帳流程，除了傳統表單的自動填寫外，還應整合Payment Request API。這個API提供了系統級別的支付資訊選擇器，用戶可以在瀏覽器原生介面中選擇已保存的支付方式和配送地址，體驗遠優於傳統表單填寫。

7. 測試跨瀏覽器自動填寫

自動填寫的行為在不同瀏覽器間存在差異。應在Chrome、Safari、Edge、Firefox等主流瀏覽器以及它們的行動版本上測試表單的自動填寫功能。特別注意Safari在Apple設備上的獨特行為——它與iCloud鑰匙圈整合，可以跨設備同步填寫資訊。
